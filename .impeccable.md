# FaceCheck — Design Context

## Design Context

### Users
ครูและผู้ดูแลระบบในโรงเรียนมัธยมศึกษาของไทย ใช้งานบน PC/laptop ในห้องพักครูเป็นหลัก รองลงมาคือ tablet หน้าห้องเรียน และ smartphone เมื่อจำเป็น ผู้ใช้ไม่ได้เป็น tech-savvy แต่ต้องการเครื่องมือที่ใช้งานได้เร็วและถูกต้องทุกครั้ง งานหลักคือเช็คชื่อนักเรียนผ่านใบหน้า ดูสถิติการเข้าเรียน และจัดการข้อมูลนักเรียน

### Brand Personality
**3 คำ**: มืออาชีพ · น่าเชื่อถือ · ฉลาด

อารมณ์ที่ต้องการ: ความมั่นใจ — ครูรู้สึกว่าระบบทำงานถูกต้องและข้อมูลเชื่อถือได้ ไม่ใช่ตื่นตาตื่นใจ แต่ต้องรู้สึกว่าใช้เครื่องมือ-ระดับมืออาชีพ ไม่ใช่แอปโรงเรียนทั่วไป

### Aesthetic Direction
- **References**: Notion (clean, spacious, typographically confident), Vercel Dashboard (data-dense แต่อ่านง่าย, minimal chrome, ใช้ negative space อย่างมั่นใจ)
- **Theme**: Light mode — ใช้ตอนกลางวันในห้องที่มีแสง
- **Palette**: Blue primary (#1A56DB) — สื่อถึงสถาบัน, ความน่าเชื่อถือ; neutrals tinted ไปทาง blue เล็กน้อย
- **Typography**: Noto Sans Thai (ต้องอ่านง่ายทุกขนาด) + Inter สำหรับ Latin + Bebas Neue เฉพาะตัวเลข display/KPI ขนาดใหญ่
- **Anti-references**: ไม่เอา emoji, ไม่เอา cartoon, ไม่เอา colorful SaaS แบบ Figma/Canva, ไม่เอา gov portal แบบเก่าที่ดูล้าสมัย

### Design Principles

1. **Data ต้องชัด ก่อนสวย** — ตัวเลขและสถิติคือ hero ของทุกหน้า; ตกแต่งมีไว้ช่วยอ่าน ไม่ใช่ดึงความสนใจออกไป

2. **Restraint เป็นความกล้า** — ใช้สีให้น้อยและมีความหมาย ใช้ white space อย่างมั่นใจ ไม่ยัดทุกอย่างให้เต็มหน้าจอ

3. **ไทยต้องอ่านได้สบาย** — Noto Sans Thai ต้อง render ถูกต้องทุก weight, line-height넉넉 (≥1.6 สำหรับข้อความยาว), ไม่ truncate คำไทยกลางคำ

4. **Feedback ที่ชัดเจน** — ทุก action ต้องมี visual confirmation ชัดเจน โดยเฉพาะ Scanner (ผลการสแกน success/error ต้องเห็นได้ทันทีจากระยะ 50cm)

5. **Progressive disclosure** — ข้อมูลซับซ้อน (Dashboard drill-down) เปิดเผยเป็นชั้นๆ อย่า dump ทุกอย่างพร้อมกัน

### Technical Constraints
- React 19 + Vite + Tailwind v4
- CSS Custom Properties system (`--fc-*` tokens) — ใช้ต่อเนื่อง ไม่เปลี่ยน
- Inline styles ผสม Tailwind (pattern ที่มีอยู่แล้ว)
- WCAG AA minimum
- No dark mode (ยังไม่จำเป็น)
- Responsive: Desktop first แต่ต้องใช้งานได้บน tablet/mobile
